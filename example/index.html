Example

You can find the full code of the components in the example below at docs/example.

This is an interactive example user interface for creating multi-send transactions.

() => {
  const [batch, setBatch] = React.useState([
    createTransaction(TransactionType.transferFunds, nanoid()),
  ])
  return (
    <ProvideMultiSendContext
      safeAddress="0x1076f084A3703E1701a1a97F837906e56370D4f9"
      network="4"
    >
      <TransactionBatch
        value={batch}
        onChange={setBatch}
        classNames={classNames}
      />
    </ProvideMultiSendContext>
  )
}

To try the example with your own Safe, simply change the values of the safeAddress and network props in the code playground above.

This example uses dnd-kit for drag & drop transaction reordering. The component styles can be customized using css modules. The example theme can be found at docs/example/plain.module.css.

For understanding the different moving parts, in the following sections we're going to break the example down into its individual components, going top-down.

<TransactionBatch>

Code: docs/example/TransactionBatch.tsx

Renders the list of <Transaction> elements, handles adding, removing, and updating transactions. Also implements the reordering of list items using the Sortable preset of dnd-kit.

<Transaction>

Represents a single transaction item in the batch. Each item has a fixed header, with a drag handle element, a title, and a remove button. Clicking the title allows collapsing the transaction.

The <TransactionTypeSelect> element allows switching between the different transaction types. When switching the transaction type, under the hood it will replace the entire transaction item with a fresh one created using createTransaction.

The <TransactionBody> component's sole responsibility is rendering the right specific component for the respective transaction type: <TransferFunds>, <TransferCollectible>, <CallContract>, or <RawTransaction>.

<TransferFunds>

Code: docs/example/TransferFunds.tsx

The component renders an <AssetSelect> (code) input, which lists all ERC20 tokens that the Safe holds a balance of. This information is provided by the useSafeBalances hook.

The same hook also powers the <AmountInput> (code) component. Here the Safe's balance of the selected token is used to control the maximum amount the user will be able to enter.

<TransferCollectible>

Code" docs/example/TransferCollectible.tsx

The most notable component here is <CollectibleSelect> (code) which uses the useSafeCollectibles hook to render a select input with all collectibles held in the Safe. Collectibles are grouped by their token contracts.

<CallContract>

Code: docs/example/CallContract.tsx

This component is powered by the useContractCall hook, which provides the contract interface information necessary for rendering input elements for all parameters of the selected contract function.
