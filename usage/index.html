Usage

Installation

This module is distributed via npm. For adding it to your project, run:

npm install --save react-multisend

To install it using yarn, run:

yarn add react-multisend

This package also depends on react. Please make sure you have it installed as well.

Transfer funds and collectibles

When building UI components for defining token transfer transactions it is useful to present a list of tokens held by the Gnosis Safe as the designated sender. The library offers two hooks to make that easy. Both hooks take the same props as an argument, defining the Gnosis Safe instance to connect to and whether to fetch the information directly or only upon request.

Props

addressstringThe address of the Gnosis Safe
networkstringThe ID of the chain on which the Gnosis Safe lives
lazybooleanWhen set to true it won't fetch until the fetch callback is invoked

useSafeBalances hook

This hook provides information about the Gnosis Safe's balances of ETH and ERC20 tokens.

const [balances, { loading, error, fetch }] = useSafeBalances({
  address,
  network,
  lazy,
})

Return value

The first item in the tuple is the list of Balance objects, one for each ERC20 token held by the Safe and one representing the ETH balance. The ETH balance is identified with null values for the token and tokenAddress fields.

Arbitrary contract calls

The package offers an additional hooks for making contract function calls leveraging the contract's interface.

useContractCall hook

This hook fetches smart contract interface information from Etherscan or the respective block explorers of other chains.

const { functions, inputs, payable, loading } = useContractCall({
  value,
  onChange,
  network,
  blockExplorerApiKey,
})

It reads and updates a CallContractTransactionInput value, which has the following fields:

{
  to: string // contract address
  value: string // amount of wei to send
  abi: string // contract interface ABI as JSON string
  functionSignature: string // function to call
  inputValues: { [key: string]: ValueType } // values to pass as input parameters with the function call
}

Props

valueCallContractTransactionInputThe contract call transaction input from state
onChange(value: CallContractTransactionInput): voidThe handler prop for updating the transaction input in the state
networkstringThe ID of the chain on which the Gnosis Safe lives
blockExplorerApiKeystring(optional) An API key for Etherscan or the block explorer for the given network

Return value

functionsContractFunction[]All state updating functions of the contract
inputsInput[]For each parameter of the currently selected function, it contains an object with type information and the input value from state
payablebooleanIndicates whether the currently selected function is payable
loadingbooleanIndicates whether the request is currently running

Encode for execution

The library supports defining transactions of four different kinds:
TransactionType.transferFundsTransfer an amount of ETH or any ERC20 token
TransactionType.transferCollectibleTransfer a non-fungible token (NFT)
TransactionType.callContractCall a verified contract by selecting a function and providing input values
TransactionType.rawAny other transaction can be defined by provided the raw encoded data

Each of these types comes with its own JSON structure for storing transaction inputs. You can find the corresponding interfaces at: src/types.ts

The TransactionInput JSON structure is not directly useful for execution but needs to be mapped to the MetaTransaction type used by Gnosis Safe to represent transactions. The library offers two functions, encodeSingle and encodeMulti, that are combined to achieve this.

encodeSingle(transactionInput: TransactionInput): MetaTransaction

The first step of encoding a list of transaction inputs into an executable format is provided by the encodeSingle function. It turns a TransactionInput object into a MetaTransaction, i.e., a representation with an ABI encoded data field.

encodeMulti(metaTransaction: MetaTransaction): MetaTransaction

The second step takes an array of MetaTransaction objects, and encodes it into a single MetaTransaction object describing the call to the MultiSend contract where the individual transactions are packed encoded into data. This is implemented in the encodeMulti function.

Example

import { isValid, encodeSingle, encodeMulti } from 'react-multisend'


if (transactionInputs.every(isValid)) {
  const multiSendTransaction = encodeMulti(transactionInputs.map(encodeSingle))
}

There are various options for executing a multi-send transaction encoded this way:

  • from any enabled Safe/Zodiac module via executeTransactionFromModule (see: Zodiac Module base contract)
  • directly by calling the Safe's execTransaction function, providing the required owner signatures (learn more)
  • collecting the required signatures on-chain, by calling approveHash upfront (learn more)
  • collecting the required signatures off-chain, by proposing the transaction using the Safe Transaction Service

Check out the @gnosis.pm/safe-core-sdk package for interacting with the Gnosis Safe contracts and the @gnosis.pm/safe-service-client package for using the Safe Transaction Service from JavaScript apps.
